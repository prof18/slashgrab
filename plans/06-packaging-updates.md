# Packaging and Updates

Slashgrab should use the same release model as Shellporter:

- Direct distribution, not Mac App Store for the first release.
- Signed, notarized, stapled `.app`.
- Zip artifact uploaded to GitHub Releases.
- Sparkle 2 automatic updates.
- Root `appcast.xml` served from raw GitHub.

Shellporter reference repo:

- `/Users/mg/Workspace/shellporter`

Observed Shellporter pieces to mirror:

- `Package.swift` includes Sparkle from `https://github.com/sparkle-project/Sparkle`.
- `Sources/Shellporter/App/SparkleUpdater.swift` wraps `SPUStandardUpdaterController`.
- `Scripts/package_app.sh` builds the `.app`, writes `Info.plist`, embeds Sparkle, signs nested frameworks, and verifies codesigning.
- `Scripts/sign-and-notarize.sh` builds universal, signs, notarizes with `xcrun notarytool`, staples, validates, and creates the final zip.
- `Scripts/make_appcast.sh` uses Sparkle `generate_appcast` to update `appcast.xml`.
- `RELEASING.md` documents the release checklist.
- `version.env` stores `MARKETING_VERSION` and `BUILD_NUMBER`.

## Slashgrab Release Identity

Production:

- App name: `Slashgrab`
- Bundle ID: `com.prof18.slashgrab`
- Zip artifact: `Slashgrab-X.Y.Z.zip`
- GitHub release tag: `vX.Y.Z`
- Appcast: `appcast.xml`

Development:

- App name: `Slashgrab Dev`
- Bundle ID: `com.prof18.slashgrab.dev`
- Must run alongside production.
- Must not consume the production update feed by accident.
- Must keep settings/history separate from production.

## Sparkle Plan

Use Sparkle 2.

App integration:

- Add SwiftPM dependency on Sparkle.
- Add `SparkleUpdater` wrapper around `SPUStandardUpdaterController`.
- Start updater after app launch.
- Enable automatic checks for production builds.
- Add `Check for Updates` in the popover/menu.

`Info.plist` keys generated by packaging:

- `SUFeedURL`
- `SUPublicEDKey`
- `SUEnableAutomaticChecks`

Recommended production feed URL:

- `https://raw.githubusercontent.com/prof18/slashgrab/main/appcast.xml`

Dev feed behavior:

- Prefer disabling Sparkle update checks in dev builds until a separate dev feed exists.
- If dev feed is added later, use a distinct `SPARKLE_FEED_URL` and signing key.

## Release Checklist

1. Bump `version.env`.
2. Update `CHANGELOG.md`.
3. Run tests.
4. Build universal app.
5. Sign and notarize with `Scripts/sign-and-notarize.sh`.
6. Create GitHub release and upload `Slashgrab-X.Y.Z.zip`.
7. Generate `appcast.xml` with `Scripts/make_appcast.sh`.
8. Commit and push `appcast.xml`.
9. Verify update feed and manual update check.

## Local Development Run Loop

Use the macOS build-run skill convention:

- Primary local run entrypoint: `Scripts/build_and_run.sh`
- Codex Run action: `.codex/environments/environment.toml`
- For SwiftPM GUI builds, stage a project-local `.app` bundle and launch with `/usr/bin/open -n`.
- Do not run the GUI app as a raw SwiftPM executable for normal verification.

All scripts stay under `Scripts/` to match Shellporter.
